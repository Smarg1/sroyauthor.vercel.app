# Contributing

Contributions are welcome. All changes must meet the project’s quality, security, and stability requirements.

## Process

1. Create a feature branch from `main`.
2. Ensure all CI checks pass:
   - Linting with zero warnings
   - TypeScript strict validation
   - CodeQL analysis
3. Submit a pull request with a clear description of changes.
4. All discussions occur within the pull request. No external channels are used.

## Standards

- Follow the existing project structure and conventions.
- Do not introduce new dependencies without justification.
- Do not commit secrets, configuration tokens, or private data.
- All contributions must remain compatible with the enforced Node version and build pipeline.

## Review

Every pull request is reviewed by the maintainer. Reviews may request changes, require simplification, or reject modifications that reduce clarity, security, or performance.
